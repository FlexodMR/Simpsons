Switching Vehicles In Game

 

Goals

 

  • A way of changing vehicles that doesn’t seem like car jacking (i.e., GTA 3)

 

Features

 

  • The user ALWAYS starts a level as a walking character (even if loading a save game).
  • A walking character can only get into a vehicle as a passenger. Therefore you will never control a vehicle that has less than two people in it.
    • If you play a Homer level, for example, you’ll never find Homer’s car – he’ll have to get a ride from other people.
    • Implication – walking characters (Bart, Lisa) don’t necessarily have to be driving characters.
  • Implication – at least some vehicles will need to be able to carry a second passenger (see Collectibles document) and the sound system will have to be able to handle three-person conversations (logic TBD in sound document)
  • The user gets out of a vehicle by pressing a button on the controller:
    • They can get out anywhere in the game world
    • The game will have to look for a “safe” exit location – a door not blocked by a static object or physics object
  • Getting into a car is done automatically by walking up to any one of a drivable vehicle’s doors (except the driver’s door)
  • Car entry/exit animation is done as follows:
    • Show the door opening
    • Camera cut combined with user instant-transportation to/from the vehicle. Don’t show character physically getting in/out of vehicles.

 

Tools

 

  • Maya world-building – designers will place car start locations for a level. They reset upon game load from the memory card.

 

Questions

 

  • Will we need to place car starts per mission rather than per level?
  • How many cars per level can we have?
