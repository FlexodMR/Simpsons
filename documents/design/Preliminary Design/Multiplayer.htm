Multiplayer

 

Goals

 

Based on a mini-post-mortem of Road Rage multiplayer, we came up with the following goals:

 

  • Keep the players close together
  • More constrained chases – keep the players from separating or getting lost during chases
  • Well hidden catch-up logic – keep the result un doubt as long as possible
  • More life through character dialog
  • Keep opponent visible (e.g., FOV, less speed, HUD, navigation system)
  • Physics (collisions, damage, and driving) tuned for multiplayer
  • Game data tuned for multiplayer
  • Improved frame rate
  • Keep it simple – fast, instant action
  • Stay true to characters
  • Cars move slower
  • Balanced chase gameplay – make sure chasing player has fair chance to catch the escaping vehicle
  • Design the game world to match these goals

 

Prioritized Features

 

  • Game Mode: Basic CTF gameplay, with variable locations on the flag but one delivery location per player per gameplay session
    • Front end screen lets you choose your character/vehicle
    • The “flag” spawns like the passengers in Road Rage
    • Each player has one destination for a gameplay session (e.g., if you are delivering to the Kwik-E-Mart, then that’s where you’ll always take the flag)
    • You steal the flag by ramming the other player
    • First one to N flags wins
    • Delivery/pickup/steal mechanism the same as last year:
      • One touch pickup/steal/delivery
      • No tag-backs
  • “Bullet Time” to create and emphasize player battlers. Components of this are:
    • slow-motion gameplay
    • fade-in transition from full speed to slow speed (and vice versa)
    • Screen blur visual effect accompanies transition
    • Game Logic/HUD, e.g.:
      • Three uses per vehicle, just like single-player powerups (the difference being that all multiplayer vehicles have only one countermeasure – bullet time)
      • User initiated with a button press
      • User’s get back a bullet time recharge (i.e., one of three uses recharges) by the following rules:
        • When the other user picks up the flag
        • When the other user steals the flag
        • When the other user is carrying the flag, and you drive for more then N seconds (tunable) and your bullet time inventory is empty
    • Countermeasure for escapee – some way to easily put the car on two wheels?
    • Tuning tools (amount of slowdown, transition time, recharge rule in code)
    • Sound – what happens in slow motion mode?
    • Camera lock – camera target is the other vehicle when in Bullet time
    • Camera – experiment with full-screen view in Bullet time
  • HUD that shows other player (e.g., arrow over their car in the game world)
  • Better way of showing who has the flag – vehicle glow or f/x, as is used in first person shooters?
  • Taunt button
  • No navigation arrow – navigate using the map, and by keeping the other player visible
    • Or do we need a crow-flies arrow or some other way of letting you know where your opponent is?
  • Damage system could be catch-up logic – person with flag more susceptible to damage
  • AI traffic as catch-up logic – more traffic (or more bad drivers) in front of flag bearer
  • Technical question – Can we use out-of-the-car gameplay as the choose-car interface? Racing to and getting in a vehicle of your choice (e.g., four cars are lined up, you have to run – possibly around objects – to your favorite)
    • Put slower cars closer to the characters start location
    • Obstacles for the player to navigate to get the best cars – race for the car mini game
    • Note that choosing a car chooses a character (driver) – so who would you be that doesn’t eliminate a driver choice? i.e., you can’t be Homer, or his car wouldn’t be available. Could use non-driver characters, like dog/cat, itchy/scratchy, etc.
    • Would this limit player vehicle choice? How many of our 16 vehicles can be in the game world at once?

 

Question

  • What are ways that we can incorporate FPS-like CTF strategies and world design?
    • Choke points in the map
    • Choke points under user control – e.g., a switch that raises a drawbridge
    • Physics-based traps you can spring on other player? Block a way?
    • Keeping the vehicles visible as much as possible – open, small world
    • Incorporate vehicle counter-measures like oil slicks?

 

 

 

Tools

 

Some of the tools designers would need to create this gameplay:

  • Maya world-building tools:
    • Place base(s) – two per level?
    • Place flag spawn-locations
    • Place drivable vehicles start locations
    • Place 2 character start locations
  • Game-tuning tools:
    • Game rules in code – revise rules based on stationary base
    • Tune counter-measures:
      • Effect (e.g., size of oil slick or smoke screen)
      • Bullet Time (amount of slowdown, transition time, recharge rate, bonus recharge values for special moves like jumping)
