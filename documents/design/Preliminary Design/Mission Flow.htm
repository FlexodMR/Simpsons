Mission Flow

 

This document describes the game flow within a level:

  • What happens when the user starts the level for the first time, or from a load game
  • What are the specific objectives they have to complete
  • How they move from objective to objective
  • How they restart missions
  • How they save the game

Goals

 

The goals of the game flow within a level are:

  • The gameplay is linear and sequential.
  • Each objective is discreet and self-contained – that is, not dependent on actions taken in other objectives.
  • The flow allows for “down time” or exploration without gameplay pressure.
  • Account for three game states – sequential completion of objectives in a game session, mission failure restarts, and loading a saved game in which some objectives are already complete.

 

Objectives

 

Definition: An objective is something the user is asked to do. It is also defined as a discreet game state. In other words, an objective is independent of anything that happened before or will happen after it. (Objectives will appear connected to the user – but an objective definition contains all the information to set up a specified situation).

 

The objectives a player can be asked to do are:

·        GO TO place

o       A broad definition of “place” will make this an extremely powerful, flexible tool. The thing you are told to go to could be:

§         A location (i.e., locator + trigger volume) laid down by designers. The door of a building (causing the apparent collection of an object). A location of a trigger in the world (that turns on an animated physics objects).

§         A character (triggers a conversation, and possibly collection of an object).

§         A vehicle (means you have to get into it – remember that vehicle location is not static).

§         An item (i.e., that you collect on foot, such as a donut – it disappears on touch).

·        COLLECT_AND_DELIVER item(s) TO destination (locator)

o       Note that an “item” can be person or object; this objective could be used, for example, to collect and deliver passengers

o       The COLLECTION and DELIVERY are part of this one objective – it is discreet, like all objectives – you’ll never physically deliver something you picked up in a previous objective.

o       The game doesn’t have to actually do anything upon collection – only delivery is the measurable objective and HUD update

·        COLLECT_AND_DELIVER item(s) FROM destination (locator)

o       Move items out of collision volume(s). Otherwise it is logically the same collect and delivery. For example:

§         Push a Duff delivery off the docks

§         Push ice-chunks out of the school doorway

·        FOLLOW AI vehicle TO destination (locator)

o       Stay within a certain distance of an AI vehicle until you reach a location

·        DESTROY object(s)

o       Objects could be AI vehicle(s) or breakables

o       AI vehicles might be on a path (laid out by designers) or using general traffic rules (roaming the streets)

 

Failure conditions for an objective can be:

  • PLAYER VEHICLE DESTROYED
  • PLAYER VEHICLE DAMAGED
  • PLAYER STRUCK BY VEHICLE WHILE ON FOOT
  • TIME EXPIRED
  • PLAYER GETS OUT OF VEHICLE???
  • FAILURE TRIGGER VOLUME – a mission or level can have a trigger volume that causes the mission to fail; this is used for “out of bounds” areas such as large bodies of water
  • Note: An objective may have no fail conditions – this is the player “down time” when they can explore, or save the game via the pause menu. You cannot save during a gameplay mission.
  • Note: The game will have a “safety valve” whereby you can skip a mission after failing it N times (designers will tune the number N). This will be recorded in your mission completion status.

 

Other attributes of an objective:

  • Maya Data for a mission (vehicle start position & trigger, AI locations, object locations, mission-specific navigation instructions, etc)
  • Dialog (could be associated with the player’s avatar, and/or other characters – logic TBD)
  • Presentation & HUD interface (e.g., camera work, NIS data, HUD text, etc.)
    • We will probably need text messages at the beginning and/or end of objectives (presentation TBD) – would this require an interface to a text bible?
    • We may want to control HUD elements – for example, a donut icon indicating that you’ve got the box of donuts
    • Camera work and presentation (e.g., you trigger a switch – the game has to show you what you’ve done)
  • AI settings:
    • Set car model, aggression, density, and speed of AI vehicle(s) globally for the objective (actual AI positions are set in Maya – density refers to AI calling for backup):

CHASE_AI POLICE EASY_DIFFICULTY MEDIUM_DENSITY

ESCAPE_AI HOMER MEDIUM_DIFFICULTY MEDIUM_SPEED

    • Set the aggression/density of the chase AI relative to how many people/objects you are carrying, e.g.:

CHASE_AI CARRYING 1 {

EASY_DIFFICULTY

EASY_DENSITY

SLOW_SPEED

}

CHASE_AI CARRYING 2 {

MEDIUM_DIFFICULTY

HARD_DENSITY

MEDIUM_SPEED

}

  • TIMER (implies a fail condition if it expires)
    • Allow for specification of SILVER and GOLD time targets. Completing a mission successfully results in a BRONZE status, and allows you to move through the game. Presentation for communicating this? Animated HUS icon? Includes information about where the next reward level is?
  • START LOCATION (a locator/trigger that kicks off gameplay)
  • VEHICLE REQUIREMENT
  • DELIVERY LOCATION
  • Bonus levels – completing a mission qualifies as “bronze” status; what are the criteria for earning silver/gold?

 

Some rules are implied by this system:

  • An objective can require a particular car, and have a triggered start location.
    • If the user is not in that car, they cannot start the mission – so the user is directed to the car instead
  • If a mission fails, the user is automatically respawned at the start point, in the correct vehicle (if there is a vehicle requirement), with physics objects, AI vehicles, and other start conditions reset
    • Implementation Note: Consider the idea of not re-spawning – the player drives back to the start location to restart a mission.
