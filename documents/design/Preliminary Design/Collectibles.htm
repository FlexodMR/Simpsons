Collectibles

 

Goals

 

  • Simple – e.g., does not increase character animation scope
  • Flexible – systems that can be re-used in many different contexts
  • Payoff – collectible events that are satisfying in some way for the user will make the game feel less linear
  • Easy to execute – simple, effective controls

 

Features – In Car

 

  • Breaking things. Driving over a specific object (e.g., Mr. Burns mailbox because it contains Homer’s embarrassing letter) or one of a set of objects (i.e., like Road Rage missions).
  • Pick up a passenger. Notes:
    • There are already two people in the car at all times. This feature means there can be three people in a car at once.
    • All vehicles should have at least 2, but ideally 3 seating locations.
    • Animation and dialog will be created for passenger-to-passenger interactions (i.e., in addition to passenger-to-driver).
  • Graphical highlight or indication of collectibles. As well as being marked on the map, collectibles need to be highlighted on-screen (e.g., by glowing, or with a floating arrow pointing at it.
  • Push N objects out of (not into) collision volume(s). Examples:
    • Push a load of Duff beer off the docks at Squidport (push an object out of a collision volume)
    • Note: The design choice that objects have to be pushed out of, and not into, collision volumes is deliberate – it is likely that pushing an object any distance with a vehicle would be frustrating

 

Features – On Foot

 

  • “Touch and disappear” – useful for small objects (a box of donuts, an election chad, etc.). Touch an object and it disappears with a sound and HUD update.
  • Graphical highlight or indication of collectibles. As well as being marked on the map, collectibles need to be highlighted on-screen (e.g., by glowing, or with a floating arrow pointing at it ala GTA3, etc.).
  • Walk up to a door and use a camera fade (plus sound effect and HUD update).
    • For example, if Homer needs a Squishee, he could be sent to the door of the Kwik-E-Mart. When he gets to the highlighted zone, there is a screen fade or camera cut to imply the passage of time where the character went in/out of the store.
  • Talk to a character - at the end of the conversation, the user gets an object (sound effect plus HUD update).
    • Could be a mission-objective (character and dialog specific to mission) or a bonus in a level (i.e., a character not shown on the HUD, that you can talk to any time in that level)
  • Flipping switches – a highlighted location in the world cause the physics animation of an object to be turned on or off (specified in a mission script). For example:
    • Press a switch that causes an ferry to start running back and forth across an area of water
    • Press a switch that starts (or stops) a drawbridge going up and down
    • Such an event might be required by a mission, or it might be an optional thing that you can do in the game world
  • Push a simple objects in the world as a trigger for a scripted event (e.g., bump a round boulder on a hill to start it moving – this triggers a pre-scripted event in the world – for example, opening a previously closed path)
  • Get into another vehicle (see “Switching Vehicles” document)

 

Design Notes

 

  • People didn’t like the simplistic “destroy N objects” missions much in Road Rage. It can be fun, but don’t overuse it.

 

Questions

 

  • What about collecting large objects (a crate of Duff beer?) or animals?
    • Will “touch and disappear” work or will it look too fake?

 

Tools

 

  • World-building:
    • Place characters for a mission, for dialog, character pickup, and/or logical object transfer (e.g., object collection might occur when you talk to someone, when you pick them up, or when you drop them off).
    • Place locators for breakable or movable object that user needs to hit:
      • Workflow notes – need a good workflow for this:
        • With naming conventions?
        • Automatic locators? (In some cases explicitly placing locators is redundant information, which leads to errors.)
          • Or automatic objects (i.e., place the locator, and the game puts an object there?)
        • Automatic trigger volumes? Manual placement of some trigger volumes that could be done procedurally.
    • Place rectangular trigger volume(s) that objects must be pushed out of. As with breakables, naming conventions should be carefully defined to avoid errors.
    • Place trigger volume(s) for passenger drop-off destinations
    • Place trigger volume(s) for “building door/camera fade” object collection (e.g., walking into the volume causes a specific presentation event  - the camera cut/fade) and the logical collection of an object.
    • Place trigger volume(s) that turn animation of a physics object on or off.
    • Place drivable vehicle locations (both mission-specific and level-specific).
