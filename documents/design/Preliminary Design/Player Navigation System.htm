Player Navigation

 

Goals

 

  • Simple and consistent - always works the same way from the user's POV
  • Error free - never points in the wrong direction
  • Flexible – allows for user choice of path when we want it to (e.g., missions with multiple pickups and user choice of the order)
  • Low learning curve for the user

 

Features

 

  • The HUD map displays one or more target icons. If it’s off-map, then it slides along the edge of the HUD map.
  • NO arrow on the HUD as in Road Rage.
  • Map icons are removed when the user picks up the item or completes the objective.
  • The pause screen will show a large version of the map as the default pause function.
  • The map will rotate to match the user’s orientation (on both the HUD and pause screen) as in Road Rage and Midtown Madness 2.
  • Targets, including moving ones such as AI vehicles, will have a highlight arrow floating over them in space. The arrow is “3D” in that it gets larger the closer you get.
  • Mission-specific navigation instructions, verbally issued from passenger to driver. For example, hit a particular collision volume and Bart, as the passenger, will say to Homer, “Turn left, Dude!”
  • General world (not mission-specific) instructions from passenger to driver (e.g., “I think that’s a shortcut”)

 

Design Notes

 

  • The first few missions in the game must be carefully designed to slowly teach the user how to navigate in the game world.

 

Tools

 

  • Designers can lay down mission-specific collision volumes that are linked to navigation instructions, for example:
    • Turn left
    • Turn right
    • Go straight
    • Turn around
    • Go up the hill
    • Go down the hill
    • Take the jump
    • There’s a shortcut
  • Designers can lay down similar information that’s always in a level, not mission-specific. This might be useful for pointing out shortcuts or other world features.
  • Each walking character will have dialog for these instructions (might be able to reuse some Road Rage dialog).

 

Questions

 

  • When the target is a moving AI vehicle, what happens when they are off-screen or behind you? Could the vehicle highlight be a beam of light for greater visibility?
    • Whatever the solution, remember that it will be used in multiplayer too.
