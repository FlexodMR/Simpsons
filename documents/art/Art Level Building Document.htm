Level Building Document – General Procedures

 

2          OVERVIEW

 

Level construction will take place with two techniques, through regular P3D Export and through Export Art Library replacement.

 

2.1        P3D Export

The regular P3D export will be used to export static world objects, this includes the track, the terrain, landmarks and generic buildings. All art though will have to pass by the art cop database though to verify that textures, vertex count, etc… are correct.

 

2.2        Export Art Library Replacement

Export Art Library Replacement will be used for dynamic props, static props, animated props and animated BV’s. Props will be exported in a file unique to that single prop and logged into the Art Library via Art Asset sheets. The location and orientation of the prop(s) in the level will be done in a prop-layout file. This props layout file will be exported to the pipeline where using the prop’s name,  the layout file will be replaced with the Art with the corresponding name in the Export Art Library.

 

2.3        Export Art Library

The Export Art Library will be a level specific collection of individual objects used for latter replacement in the pipeline. For every object there will be a unique maya and p3d file.

 

2.4        Art Database

All art that will pass into the game must be registered in the Art Database. The Art Database will track vertex counts, poly counts, textures, shader flags etc… The Art Cop will be the process by which art will pass or fail on it’s way through the pipeline. Art may be grouped together under a common prefix name and texture collection. Ie for all the generic buildings in L1 zone A, you might register the art database to accept any mesh named “L1A_gen*” – (* being a wildcard) and textured with the maps brick_pnkA.bmp, brick_pnkB.bmp, brick_pnkC.bmp, brick_pnkD.bmp and have less than 100 vertex (some general failsafe condition). So, any generic building will pass the art database if it’s named with the L1A_gen prefix, has less than 100 vertexes and is textured with the brick_pnkA.bmp map.

 

3          TERMINOLOGY 

 

TRACK  - Driveable ground surface incorporating the collisions mesh, landmarks footprints, parks, features etc. Constructed from an 8m pre-mapped units.

 

LANDSCAPE – Non-Driveable ground surfaces, excluded from the collision map constructed from 8m pre-mapped units.

 

LANDMARKS– Higher Poly, High map features with driveable footprint surrounding them (that will be later incorporated into the track mesh). Landmark footprints are constructed generally to accommodate the 8m base units, but should not be restricted to them. At the footprints perimeter the 8m units should be accommodated.

 

DYNAMIC PROPS – Moveable or Disappearing objects

 

STATIC PROPS – non-moving or disappearing multiple occurring objects

 

ANIMATED PROPS – vertex animating or texture animating objects

 

ANIMATED BV’s – vertex and bounding volume animation controlled by triggers

 

STATIC OBJECTS – the bulk of static world building objects.

 

3.1        HIGH LEVEL FEATURES 

 

3.1.1          TRACKS

·        Base track units of 8m. Typical track subdivision of 2 x 3m driving lanes and 1 x 2m shoulders. Units are typ. ½ of road width.

·        Base Shoulder units of 8m. Typical shoulder subdivision of 2m sidewalk/grass, 2m sidewalk/grass and 4m lawn

·        Unitized track textures drawn to consistent scale of 8m units, and base layout

·        Predetermined angles - 15° snapping

·        Predetermined sidewalk curbs 1:2 slope - .1m rise over .2m run – typical sidewalk width of 2m

·        Preset max slopes - 45° - 1:1 slope max

·        Track is deformed using Maya wires for easy iteration

3.1.2          LANDSCAPE

·        Base 8m unit

·        Unitized track textures drawn to consistent scale of 8m units, and base layout

·        Landmark/Track do not always need to be resolved at edges as a physical barrier between drivable and non-drivable surfaces will be present (fences, buildings, etc…)

 

3.1.3          DYNAMIC PROPS

·        Dynamic props are created to scale (meters) in a unique prop file with 1 prop (with possible multiple meshes) per file

·        Props are built at the origin, with their center point at the origin and should located in the bottom, left hand corner of the mesh(s).

·        This singular prop will be exported to the prop library unique to each level. The prop will be named and registered on an Art asset sheet.

·        Textures for the prop will be located in the associated prop source images directory and registered on the Art asset sheet.

·        Location and position of props will be done in a prop layout and replaced in code using name and the Art Library

·        Attributes will be associated with the prop in the Art Database.

 

3.1.4          STATIC PROPS

·        Static props will be created to scale (meters) in a unique prop file with 1 prop (with possible multiple pieces_ per file

·        Props are built at the origin, with their center point at the origin and should located in the bottom, left hand corner of the mesh(s).

·        This singular prop will be exported to the prop library unique to each level. The prop will be named and registered on an Art asset sheet.

·        Textures for the prop will be located in the associated prop source images directory and registered on the Art asset sheet.

·        Location and position of props will be done in a prop layout  and replaced in code using name and the Art Library

·        Attributes will be associated with the prop in the Art Database.

 

3.1.5          ANIMATED PROPS

·        Animated props are created to scale (meters) in a unique prop with 1 prop (with possible multiple meshes) per file

·        Props are built at the origin, with their center point at the origin and should located in the bottom, left hand corner of the mesh(s).

·        This singular animated prop will be exported to the prop library unique to each level. The prop will be named and registered on an Art asset sheet.

·        Textures for the prop will be located in the associated prop source images directory and registered on the Art asset sheet.

·        Location and position of animated props will be done in a prop layout  and replaced in code using name and the Art Library

·        Attributes will be associated with the prop in the Art Database.

 

3.1.6          PROP LAYOUT

·        Location and position of all props (dynamic, static and animated) will be done in a prop layout file

·        Naming conventions in these files are critical as named objects will be replaced by the same named objects that exist the Prop Library

·        Prop name based replacement will occur in the pipeline

·        Props will be imported to the prop layout file from the prop directory and then duplicated to their locations, originals will remain at the origin and hidden, used only for reference. (layout objects may also be simple placeholders or abstract low poly representations of the props)

·        Prop updating and replacing in the layout file will be done with the copyToInst.mel

·        Grounding of the props will be achieved with the referencing the track and landscape files and using objectSnap.mel

 

3.1.7          LANDMARKS

·        Built to a 4m base construction unit (the 4 m grid is only a starting point, if it would be more efficient to use a larger scale or break from the grid, do so after advising the Art Director and Lead Artist)

·        Maps designed to 4m (128x128 pixel max.) scale consistent with construction unit allowing for consistent resolution and proportions (deviate when necessary as above…)

·        Footprint built from base track units but customized to address landmark requirements. Footprint maps should re-use ground textures where possible.

·        Footprint edge should conform to TRACK 8m base units.

·        Landmarks will be constructed in a unique landmarks sub-folder and then placed in the landmarks layout file. Static landmark buildings will be treated as static objects.

·        All meshes that belong to the landmark should be grouped and named specifically with a name that will be registered on an Art record sheet .

·        Textures that are associated with the Landmark will be located in the corresponding folder in Source Images and registered on an Art record sheet for entry in to the Art Database.

·        Dynamic Props and Animation associated with the Landmark should be constructed in the Landmark file, but the copied into a unique file in the level’s prop folder and registered with an Art Asset Sheet of their own.

 

3.1.8          STATIC OBJECT

·        Built to a 4m base construction unit

·        Maps should conform to a unitized tile able texture set consisting typical building components (e.g. Base material, door, windows, upper windows…). This will allow for a maximum amount or re-use, multiple arrangements

·        Static props should be constructed to similar 4m base units so that they may easily be arranged on generic prop “base”

·        Static prop sets should be given a general Art Asset sheet that records naming convention, suggested max vertex counts, and map sets.
